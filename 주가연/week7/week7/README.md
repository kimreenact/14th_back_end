1. 각 계층이 어떤 역할을 하는가
   1. Controller: 클라이언트의 HTTP 요청을 가장 먼저 처리하는 계층 
   2. Service: 비즈니스 로직을 처리하는 계층으로 DTO에 담긴 데이터를 꺼내 Menu를 생성하고 Repository를 제어 
   3. Repository: 데이터 액세스 계층으로, 메모리를 활용하여 데이터를 CRUD하는 역할 
   4. DTO: 포스트맨을 통해 JSON 형태로 보낸 요청 데이터를 받아내기 위한 전송 전용 객체

2. MVC와 DI/IoC가 어떻게 적용되었는가
   1. MVC 패턴: Controller가 데이터를 가공한 뒤 JSON으로 데이터를 반환하여 POSTMAN에서 화면을 자유롭게 그리도록 돕습니다. 
   2. DI: 클래스 내부에서 직접 의존 객체를 생성하지 않고, 외부에서 객체를 주입받음 
   3. IoC: 어노테이션을 사용하여 각 클래스를 Spring 컨테이너의 관리 대상으로 등록
