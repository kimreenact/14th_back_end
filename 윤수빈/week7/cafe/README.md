# 메가커피 그만두고 만든 개인카페
---------
## 1. 각 계층이 어떤 역할
* Controller: 문지기처럼 인터넷 주소창을 감시하다가 사용자가 링크를 누르거나 주소를 치고 들어오면 `Service`한테 작업 보내고 어떤 화면을 보여줄지 정해줌.
* Service: 약간 우두머리 느낌 Controller한테 요청을 받으면 Dto를 진짜 메뉴판 객체로 새로 조립하는 등 여러가지 처리
* Repository: 데이터가 저장되고 관리될 수 있도록 넣고 뺼 수 있게 함
---
## 2. MVC와 DI/IoC가 어떻게 적용
* MVC 패턴: `Controller`가 요청을 처리한 뒤 데이터를  `Thymeleaf HTML`에 전달하도록 구현함
* DI(의존성 주입): `MenuService`와 `MenuController`에서 생성자 주입 방식으로 `@RequiredArgsConstructor`를 사용하여 필요한 객체만 주입가능하게 함
* IoC(제어의 역전): `@Controller`, `@Service`, `@Repository`어노테이션을 사용하여 객체를 직접 생성하지 않고 Spring이 자동으로 객체를 생성하고 관리하게함
---
## 3. 느낀점
### 어려웠던 점
* 기능 하나를 구현하는데 `Controller`, `Service`, `Repository`등 여러 파일이 유기적으로 연결되어 있으니까 힘들었음
* 수정 기능 구현할 떄 데이터를 화면에 출력하고, 수정된 값을 다시 DTO로 받아 원본 데이터를 변경하는 것과 `Optional`과 `redirect`의 흐름을 따라가는 것이 까다로움
### 더 구현해보고 싶은 기능
* 메뉴 삭제/이미지 업로드/정렬/검색 기능
* 실제 DB(MySQL) 연동: 서버가 꺼져도 데이터가 보존되게 하는 기능
* 관리자 로그인 기능: 일반 손님은 조회만. 매니저는 메뉴를 등록 및 수정할 수 있는 권한 보안 기능
### 스프링을 쓰면서 편리했던 점
* lombok의 매크로가 있어서 길게 쓰는 코드를 생략 가능하다는 점이 어려웠지만 익숙해지면 편리하다고 생각함
